# Security Policy

## Supported versions

The latest published release receives security fixes.

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Use GitHub private vulnerability reporting when it is enabled for this repository.

Include:

- a description of the vulnerability
- affected commands or files
- reproduction steps
- potential impact
- suggested mitigation when available

Do not include real credentials, private keys, tokens or customer data.

## Scope

Security concerns may include:

- command injection
- unsafe repository handling
- secret disclosure
- unsafe Git operations
- unsafe Composer options
- unexpected production changes
- automatic merging or deployment
- insecure temporary-file handling
